# DESIGN

## 项目目标

`liruijian.com` 不应该长期停留在“单个项目官网”的形态，而应该成为：

- 个人主页
- 作品库入口
- 可体验应用入口
- 方法笔记与观察的发布地

当前版本是 `Personal Site V1`，重点不是一次性做全，而是先把结构做对。

## 根域首页的职责

首页只承担四类任务：

1. 用一句话解释“你是谁、你在做什么”
2. 让访客快速进入作品和应用
3. 暴露你的方法论与判断，而不是只暴露项目截图
4. 给未来扩展多项目、多应用、多文章留下清晰骨架

这意味着首页不能继续被某一个活动、训练营或黑客松长期占据。

## 信息架构建议

推荐的长期结构：

- `/`：首页，建立认知与分流
- `/projects`：完整作品列表
- `/projects/slug`：单个作品详情
- `/apps`：可体验应用列表
- `/apps/slug`：单个应用介绍页
- `/notes`：文章、复盘、观察
- `/about`：个人介绍与合作方式

当前 V1 先在首页中用分区实现这些栏目逻辑，后续再拆成真正多页面结构。

## 内容层级

首页按以下顺序组织：

1. Hero
   说明站点定位，给出 2 到 3 个主入口。
2. 结构建议
   解释为什么首页、作品、应用、Notes、关于要分层。
3. 精选作品
   放最值得点开的真实项目。
4. 可体验应用
   放用户能直接体验的产品入口。
5. Notes
   放方法型内容，建立持续更新感。
6. 关于
   放站点路线、视觉原则和下一步建议。

## 视觉方向

整站走“编辑感 + 展柜感”路线，而不是“科技大会海报风”。

- 背景：暖灰纸面
- 主文字：深墨色
- 强调色：砖红、墨蓝、少量金色
- 气质：克制、结构清晰、有审美但不过度装饰

这样更适合：

- 长期做个人品牌
- 同时承载项目、应用和文章
- 避免看起来像一次性的活动站

## 为什么保留黑客松页

原站点并不是无效资产，它更适合变成：

- 一个代表作品详情页
- 一个可在首页 `作品` 栏被引用的案例
- 以后 `/projects/...` 结构的先行样板

所以本次没有删除，而是下沉成 `hackathon.html`。

## V1 的边界

当前版本已经完成：

- 根域首页切为个人站
- 作品 / 应用 / Notes / 关于 四段结构
- 现有黑客松项目的归档保留
- 视觉风格从项目页切换为长期个人站风格

当前版本暂未完成：

- 真正的多页面路由
- 更多真实项目详情页
- 真正可体验的 live apps
- 真实联系方式与社交入口
- Notes 正式内容页

## 后续建议

下一轮最值得补齐的是内容，不是继续堆样式：

1. 补 2 到 3 个真实项目详情页
2. 放 1 到 2 个真实可体验应用
3. 写 3 篇短 Notes
4. 再拆多页面结构

先有内容密度，再做页面扩张，站点会更稳。
