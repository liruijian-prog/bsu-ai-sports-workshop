# Workstation Checklist

本文件用于会前完成 AI 工作台准备，确保“每位学员一台电脑”的实战工作室可以顺利开展。

## 1. 每台电脑最低基线

- 可正常联网
- 浏览器最新版
- VS Code 或同类编辑器
- Git
- Node.js 20+ 或当前课程指定版本
- Python 3
- 可正常打开命令行终端

## 2. 核心工具清单

- Claude Code / CC
- Codex CLI
- Gemini CLI
- 至少一种国产大模型入口或 API
- 演示所需浏览器插件或开发辅助插件
- 项目模板、案例包、提示模板

## 3. 账号与 token 策略

需要先确定：

- 学员是否自备账号
- 主办方是否统一发放测试 token
- 国际工具和国产工具分别采用什么账户策略
- 现场是否准备备用账号
- 如何在课后回收、禁用或更换临时 token

## 4. 网络与合规

- 测试教室网络是否稳定
- 明确是否允许学员连接外部热点
- 如使用校园网，确认是否需要域名白名单
- 明确“不提供翻墙 VPN 服务”的现场口径
- 对涉及真实数据的演示做脱敏处理
- 所有案例仅用于教学科研或内部演示

## 5. 会前时间表

### T-7 到 T-5

- 锁定软件版本和安装说明
- 生成统一项目模板
- 准备 20 个场景案例包
- 准备国内替代方案和离线备用素材

### T-4 到 T-2

- 在实际场地做网络与投影测试
- 抽样验证 Windows 和 macOS 的安装流程
- 验证 CC、Codex CLI、Gemini CLI、国产模型路径是否都可运行
- 准备常见报错处理清单

### T-1

- 完成全流程彩排
- 确认签到物料、名牌、电源插排、延长线
- 确认每位教练和助教的站位与职责
- 准备 U 盘或本地拷贝作为最后兜底

## 6. 现场支持建议

- 每 20 到 25 名学员至少配置 1 名技术助教
- 入口处准备“安装失败 / 登录失败 / 网络失败”的快速分流机制
- 现场准备一台备用电脑用于临时替换
- 投影机和主讲电脑要单独提前 30 分钟调试

## 7. 建议随手可发的通知文案

### 学员开班前通知

请自带可正常联网的笔记本电脑，提前确认浏览器、编辑器、Git、Node.js、Python 3 可正常使用，并准备好个人体育场景需求。课程将涉及 CC、Codex CLI、Gemini CLI 及国产大模型应用路径。本研讨班不提供翻墙 VPN 服务，相关国际工具需在合规前提下自行具备可用条件。

### 助教会前通知

请在开班前一天完成所有设备、网络、账号和案例包测试。现场优先保障签到、安装排障、网络问题和投影切换，所有异常统一登记，避免重复排障。
