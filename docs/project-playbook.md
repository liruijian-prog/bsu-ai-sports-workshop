# Project Playbook

## 1. 项目目标

把本项目从“招生简章样稿”推进成一次真正可落地的 `AI+体育 Vibe Coding 训练营暨创新黑客松`，最终交付以下结果：

- 1 个可对外发布的项目网站
- 1 套完整的报名承接与咨询转化流程
- 1 套 `2 天训练营 + 7 天竞赛 + Demo Day` 的项目机制
- 1 套现场 AI 工作台与冲刺期模板体系
- 20 个可同时用于培训与竞赛的体育命题
- 1 套公开提交标准、评分标准与评审组织方案
- 1 套赛后优秀项目孵化和合作线索跟进机制

## 2. 推荐推进原则

- 先审批，后公开：未确认机构名义、合作边界和标准落款前，不建议对外发布。
- 先定赛制，后招生：时间表、奖项、评审和提交规则不清楚，招募会严重失真。
- 先演练，后开营：所有工具、账号、模板和现场流程必须至少完整彩排 1 次。
- 先保证最小闭环，再做加分项：优先保证网站、报名、训练营、竞赛和 Demo Day 全链路跑通。

## 3. 阶段路线

### Phase 01: 立项与合作口径确认

建议时间：`T-35` 到 `T-24`

核心动作：

- 明确项目是否可以正式以学院名义对外发布
- 确认主办 / 承办 / 协办关系和标准落款
- 确认项目是否明确写“借鉴 Google Gemini Vibe Coding Hackathon 的 sprint 机制”
- 确认训练营、竞赛和 Demo Day 的组织方式、奖项和预算
- 确认导师、评审和合作方的可参与边界

本阶段交付物：

- 对外发布口径确认稿
- 项目基本信息表
- 初版预算与资源清单

阶段门槛：

- 如果审批口径未确认，后续所有公开内容都应保持“样稿”状态

### Phase 02: 内容与赛制定稿

建议时间：`T-23` 到 `T-15`

核心动作：

- 定稿项目网站文案、两天日程、FAQ、收费政策与组队说明
- 复核 20 个命题是否既适合训练营带练，也适合竞赛阶段继续拉开差距
- 定义提交物：公开链接、视频 demo、项目说明、可选附录
- 定义评分标准：Impact、Technical Depth、Creativity、Presentation
- 为导师 / 评审补齐姓名、职务、擅长方向、照片与代表案例

本阶段交付物：

- 网站对外版本文案
- 两天训练营讲义与案例包目录
- 竞赛规则、评分表与提交流程说明
- 导师 / 评审简介卡

阶段门槛：

- 内容和赛制没有定稿前，海报、报名表单和咨询话术不要提前扩散

### Phase 03: 招募上线与赛前预热

建议时间：`T-14` 到 `T-7`

核心动作：

- 发布网站、海报、二维码、报名表单和组队说明
- 设置咨询联系人、咨询话术、优惠审核规则和缴费路径
- 组织线上说明会，完成工具自检和赛前答疑
- 建立报名台账、参赛队台账和导师 / 评审沟通台账

本阶段交付物：

- 上线版网站与二维码
- 报名表单与咨询脚本
- 组队与命题预匹配清单
- 招募日报与跟进机制

阶段门槛：

- 如果支付、开票、组队和联系人口径不清晰，报名转化会严重受阻

### Phase 04: 训练营开营与 Sprint Launch

建议时间：`T-6` 到 `T0`

核心动作：

- 完成 AI 工作台环境安装、账号测试、模板包测试与网络测试
- 按学员数量准备电脑、插排、投影、讲师屏、签到与名牌
- 组织至少 1 次全流程彩排，模拟 Day 1 到 Day 2 的关键节点
- 在 Day 2 结束前完成 MVP、公开链接、视频脚本和 sprint backlog 冻结

本阶段交付物：

- 可运行的统一工作台
- 训练营现场物料清单
- 参赛队冲刺计划表
- 导师 office hours 排班表

阶段门槛：

- 国际工具和国产替代路径都必须演练成功，确保现场不会因单点失效停摆

### Phase 05: 冲刺赛、决赛路演与赛后转化

建议时间：`T+1` 到 `T+10`

核心动作：

- 组织 office hours、中期检查和最终提交通知
- 汇总公开链接、视频、说明页和评分材料包
- 完成 Demo Day、评审打分、奖项公布和优秀作品沉淀
- 会后 48 小时内发资料包、回访问卷、合作意向登记
- 对重点单位推进二期课程、试点项目或系统原型孵化

本阶段交付物：

- 作品清单与评分结果
- 媒体素材包与优秀案例库
- 回访记录与合作线索池
- 二期推进建议

阶段门槛：

- 会后没有在 48 小时内跟进，项目热度和转化率通常会明显下降

## 4. 最终交付清单

- 项目网站
- 报名二维码 / 表单
- 海报或转发长图
- 两天训练营讲义与案例包
- 20 个体育命题模板
- 提交规则、评分标准与评审材料包
- 导师 / 评审简介页
- 现场执行手册
- 赛后资料包、问卷与孵化清单

## 5. 核心风险

- 名义使用和“借鉴 Google”表述未统一
- 时间地点已对外释放，但训练营与 Demo Day 资源尚未锁定
- 赛制可讲，但提交模板、评分表和评审组织没有提前统一
- 工具可演示，但学员电脑现场跑不起来
- 训练营交付了，但冲刺赛没有中期校准，团队迅速失焦
- 报名线索有了，但支付、开票、咨询承接和组队机制未闭环

## 6. 建议工作节奏

- 每周 1 次项目例会，聚焦进度、风险、待决事项
- 每个阶段结束时做一次 gate review
- 所有待补字段统一记录到 `docs/info-intake.md`
- 所有任务和截止时间统一记录到 `docs/task-board-template.csv`

## 7. 推荐下一步

如果现在就开始推进，建议先做四件事：

1. 补齐机构审批、训练营时间、竞赛时间线、奖项和联系人口径。
2. 锁定导师 / 评审、20 个命题和评分标准的最终边界。
3. 组织一次 60 分钟的工作台与提交流程演练。
4. 对外文案统一写清“借鉴 Google Gemini Vibe Coding Hackathon 的 sprint 机制，但不是 Google 官方赛事”。
